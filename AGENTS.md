# Esora Maps

架空のミニチュア交通マップ。**HTML / CSS / JS + Canvas 2D のみ**で動く自己完結アプリ。
サーバ・外部データ・ビルド・依存ライブラリは無し。

## 壊してはいけない前提
- **道路部品の接続契約**: 接続口は共有エッジの純関数（map.js の `portV`/`portH`）。隣接部品は必ず一致 → チャンク境界で道路が途切れない。
- **車両は通行パス上だけを走る**（`roadpart.lanePath`）。道路外に出さない。
- **交差点はデッドロックフリー**: クラスタ予約（`map.clusterKey`）は「踏み込む瞬間」にだけ取得する（接近中に握らない＝hold-and-wait 禁止）。
- **左側通行**。
- **タップは限定インタラクションのみ**：路肩のゴミ／郵便物（郵便ポスト上）をタップするとそれが**ハイライト**され（ゆっくり跳ねる＝特別車の目的地）、対応する特別車（ゴミ→収集車・郵便物→郵便車）がそこへ向かう（不在ならスポーン）。複数種が近接していたら**最寄りの収集対象に振り分け**（ハイライトは種別ごと独立）。タップが効いた合図として**共通の視覚エフェクト（波紋）を出す**（effects.js。特定オブジェクト固有でなくアプリ共通）。同種の別オブジェクトをタップするとハイライトはそちらへ移る。それ以外のタップは何も起こさない。**詳細表示・投稿・評価・検索・ログイン・現在地・外部遷移は引き続き禁止**。タップが「消費」されなければダブルタップズームは従来どおり。
- **路肩に寄せ切った（aside）車両は移動できない**（vehicles.js。寄せる途中の徐行は可）。→ 収集車／郵便車は「回収できる距離」で寄せて停止し回収する／確保車・道を譲る車も停止。
- **収集車／郵便車はハイライトされた対象だけを回収**：目的地が無ければ探索も回収もせずただ走る（反応的な自動回収はしない）。回収対象は**同じ直線道路の両車線**のみ（走行車線＋対向車線。対向の対象も自車線の路肩へ寄せて**同じ沿道距離**で回収する／横の判定で**並行する別の道路**の対象は対象外＝そちらへ向かって走り続ける）。路肩へ寄せ切ってから**約1秒後**に回収し、両車線に同時にあれば両方回収する。この収集挙動は汎用エンジン（scenario.js の `makeCollectionEvent`/`spawnCollector`、収集対象は collectible.js のプール）で、ゴミ収集と郵便回収が同一の仕組みを共有する。
- **郵便**：郵便ポストはバス停のようにまばらに配置（map.js の `tile.post`。視覚衝突回避でバス停タイルとは排他）。郵便物はそのポスト上にだけ稀に湧く（mail.js）。タップ→ハイライト→**赤い郵便車**（`role:'post'`・同時1台・チェイス/ゴミ収集とは別種なので共存可）が向かい、上記の収集挙動で回収する。郵便物の座標はポスト上だが、見た目は**ポストへの吹き出し**＋ハイライトは**ポストを囲むリング**（render.js）。
- 起動は **HTTP 配信が必要**（ES モジュールは `file://` 不可）。PWA インストール / Service Worker は **HTTPS か localhost** が前提。

## マップ生成の決定論
- 乱数源は rng.js の `hash(x,y,salt)` のみ。**map.js / rng.js では `Math.random` を使わない**。
- `SEED`（config.js）は**読み込みごとにランダム・セッション中は固定** → スクロールで離れて戻ると同じ地形、リロードで別の街。環境変数 `ESORA_SEED` で固定可。
- `Math.random` は交通（vehicles.js）・出来事（scenario.js）・路肩のゴミ（litter.js）・郵便物（mail.js）だけ → これらはリロードで変わる。バス停と郵便ポスト（map.js）は決定論側（`hash`）なのでリロードしても同じ。

## ディレクトリ構成（web ルート = リポジトリのルート）
- ルート — `index.html` ＋ PWA 設定（`manifest.webmanifest` / `sw.js` / `icon-*.png`）。配信/デプロイのエントリ。
- `src/main/` — アプリ本体（ES モジュール）。`index.html` が `src/main/main.js` を読み込む。
- `src/test/` — ヘッドレステスト。**PWA には含めない**（Service Worker が ASSETS と実行時キャッシュの両方で `src/test` を除外）。

## モジュール構成（src/main/。責務分割で、main.js は初期化と接続のみ）
- config.js — 定数・方向テーブル・`SEED`
- rng.js — 決定論ハッシュ
- roadpart.js — 道路部品の契約（接続口＋通行パス）＋沿道オブジェクト位置（`shoulderPoint`。ゴミ/ポスト/郵便物が同じ路肩点に乗る共有幾何）
- map.js — 道路網生成・接続トポロジ・バス停・郵便ポスト（`tile.post`。まばら・バス停と排他）・交差点クラスタ
- vehicles.js — 車両モデルと走行ルール（追従減速・交差点予約・バス停車・路肩退避）
- collectible.js — 収集可能オブジェクトのプール（配列管理・最寄り探索・範囲回収・ハイライト）。litter / mail が共有
- litter.js — 路肩のゴミ（車両以外のワールドオブジェクト。生成のみ持ち、回収/ヒットテスト/ハイライトは collectible へ委譲）
- mail.js — 郵便物（郵便ポスト上にだけ湧くワールドオブジェクト。生成のみ持ち、回収等は collectible へ委譲）
- effects.js — アプリ共通の一時的な視覚エフェクト（タップ波紋。寿命管理のみ。描画は render）
- camera.js — ビュー変換・ズーム・画面↔ワールド変換
- render.js — 描画（道路・バス停・郵便ポスト・車両（赤い郵便車含む）・ゴミ／郵便物（ポストへの吹き出し）＋ハイライトの跳ね・波紋エフェクト ＋ 特別車の位置インジケータ＝画面外は端のアイコン／画面内は車両に白い透過ハイライト[全車種同じ大きさ]。収集車/郵便車は目的地へ向かう間だけ。引きで車両/ゴミ/郵便物を拡大）
- input.js — ドラッグ / ピンチ / ホイール / ダブルタップ / タップ（onTap）
- scenario.js — ScenarioEvent 基盤 ＋ カーチェイス（逃走車 vs パトカー）＋ 収集の汎用エンジン（`makeCollectionEvent`/`spawnCollector`）でゴミ収集（収集車 vs ゴミ）と郵便回収（郵便車 vs 郵便物）。タップでハイライト誘導・最寄りの対象へ振り分け。別種は同時発生可（チェイス/ゴミ収集/郵便が共存しうる）・同種は単一
- PWA 設定（ルート）: index.html ＋ manifest.webmanifest / sw.js / icon-*.png

## 規約
- コメントは日本語、既存スタイルに合わせる。
- 依存追加・ビルド導入はしない。Canvas 2D のみ。

## 動作確認
- ローカル表示: `tools/serve.sh [PORT]`（ルートを HTTP 配信しブラウザを開く。停止は Ctrl-C）。
  中身は `python3 -m http.server`（ルート配信）相当。
- ヘッドレステスト（自己完結・依存なし。どのシードでも通る）:
  `node src/test/esora-mod-test.mjs`
  接続口一致/行き止まりゼロ/部品語彙/バス停率・通行パス契約・走行の不変条件
  （道路逸脱/重なり/デッドロック無し）・チェイス・確保・アイコン・スポーン方向・決定論・
  路肩退避（aside は移動不可）・ゴミ収集（タップした目的地へ向かい寄せて回収・操作なし＝ハイライト無しでは
  探索/回収しない・対向車線も同じ沿道距離で回収・タップ誘導/スポーン）・ハイライト/タップ波紋・
  郵便（ポストのまばら配置＋バス停と排他・郵便物はポスト上にのみ湧く・タップ誘導/回収/対向/振り分け・郵便車アイコン/吹き出し描画）・描画を検証。
